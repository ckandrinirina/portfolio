# Story 02-06: Card component

> **Epic:** Design System — UI Primitives & Hooks
> **Size:** S
> **Status:** TODO

## Description

Create `src/components/ui/Card.tsx` — a surface shell used by the `Experience`
and `Projects` sections to visually group related content. The `Card` component
is intentionally a blank canvas: it provides the visual container (border,
background, optional shadow/elevation) and leaves all internal layout to its
`children`. Both Experience and Projects will render arrays of `Card` components.

## Acceptance Criteria

- [ ] `Card` renders its `children` inside a single block-level element (e.g. `<div>` or `<article>`).
- [ ] The rendered element has a visible border or elevated background that clearly separates it from the surrounding page surface in **light mode**.
- [ ] The same visual separation is maintained in **dark mode** — border colour and/or background token updates when `.dark` is applied.
- [ ] The card has rounded corners consistent with the design token (e.g. `rounded-lg` or `rounded-xl`).
- [ ] An optional `className` prop is accepted and merged via `cn()`.
- [ ] Passing `className` does not suppress the base card styles.
- [ ] `children` accepts any React node.
- [ ] The component is a default export from `src/components/ui/Card.tsx`.
- [ ] TypeScript compiles without errors; `children: React.ReactNode`, `className?: string`.
- [ ] ESLint reports no violations.

## Technical Notes

- Use semantic tokens from Story 02-02 (e.g. `bg-surface-elevated`, `border-border`) so dark-mode colours are inherited automatically from CSS custom properties.
- Using `<article>` as the root element is semantically appropriate when the card wraps a self-contained experience or project entry. If the consuming component (Experience/Projects) needs more control over the landmark element, `<div>` is also acceptable — note the trade-off in the implementation.
- Example base classes: `rounded-xl border border-border bg-surface-elevated p-6 shadow-sm`.
- Do not add hover or interactive states here — `Card` is a display container, not a link or button.
- Padding inside the card (`p-*`) should match the design; if uncertain, use `p-6` as a starting point and let consuming stories adjust via `className`.
