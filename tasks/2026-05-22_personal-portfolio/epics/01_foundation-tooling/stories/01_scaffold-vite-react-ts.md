# Story 01-01: Scaffold Vite + React 19 + TypeScript project

> **Epic:** Project Foundation & Tooling
> **Size:** S
> **Status:** TODO

## Description

Run `npm create vite@latest . -- --template react-ts` in the repository root to establish the baseline project files (package.json, vite.config.ts, tsconfig.json, tsconfig.node.json, index.html, src/main.tsx, src/App.tsx), then install all runtime and dev dependencies. The repository already contains a `docs/` directory that must be left entirely intact. This story creates the structural skeleton every other story and epic builds on.

## Acceptance Criteria

- [ ] `package.json` lists `"react": "^19"` and `"react-dom": "^19"` as runtime dependencies.
- [ ] `package.json` lists `"typescript": "^5.7"` (or a patch of 5.7+) as a dev dependency.
- [ ] `package.json` lists `"vite": "^7"` as a dev dependency.
- [ ] `tsconfig.json` has `"strict": true` under `compilerOptions`.
- [ ] `tsconfig.json` has `"jsx": "react-jsx"` under `compilerOptions`.
- [ ] `tsconfig.json` has `"moduleResolution": "bundler"` under `compilerOptions`.
- [ ] `tsconfig.node.json` exists and is referenced in `tsconfig.json` `references`.
- [ ] `index.html` exists at the project root and contains a `<div id="root">` mount point.
- [ ] `src/main.tsx` mounts `<App />` (or the scaffold equivalent) into `#root`.
- [ ] `src/App.tsx` exists as the top-level component.
- [ ] `npm install` completes without errors; `node_modules/` is populated.
- [ ] `npm run dev` starts the Vite dev server on `http://localhost:5173` and the scaffold UI is visible in a browser without console errors.
- [ ] The `docs/` directory (including all subdirectories and files) is unchanged after running the scaffold command â€” no overwrite, no deletion.
- [ ] No `.eslintrc.*`, `postcss.config.*`, or `tailwind.config.*` files are created by this story (those belong to subsequent stories).
- [ ] `vite.config.ts` is present and imports `defineConfig` from `'vite'`; it includes the `@vitejs/plugin-react` plugin (scaffolded default).

### Edge Cases

- Running `npm create vite@latest` in a non-empty directory: the CLI may prompt "Current directory is not empty. Please choose how to proceed." â€” select the option to keep existing files (or use `--force` only if it preserves non-conflicting files). Verify `docs/` is untouched afterwards by listing its contents.
- If the scaffold generates a `.gitignore`, confirm it does not conflict with any existing `.gitignore` already in the repo root.
- TypeScript version: the template may install a TypeScript version older than 5.7; after scaffolding, upgrade with `npm install -D typescript@^5.7` if needed, then confirm `tsc --version`.

## Technical Notes

- Command: `npm create vite@latest . -- --template react-ts` (the dot targets the current directory).
- After scaffold, run `npm install` to populate `node_modules/`.
- The `@vitejs/plugin-react` plugin is included in the template by default; do not remove it â€” later stories add plugins alongside it.
- Do not configure Tailwind, ESLint, Prettier, or Vitest in this story; those are handled by stories 01-02, 01-03, and 01-04 respectively.
- `src/App.css` and `src/assets/react.svg` may be generated by the scaffold; leave them for now â€” they will be cleaned up or replaced by later stories in epic 02.
